macOS 没有暴露的桌面模式

不是调暗亮度。不是黑色遮罩。不是合盖模式。

Clamless 面向那些希望 MacBook 开盖放在桌面上，但不想让窗口、鼠标或显示器排列状态跑到内置屏幕上的用户。

01

断开系统布局

内置显示器会从 active WindowServer arrangement 中移除，所以鼠标不会进入内置屏幕。

02

关闭物理面板

布局断开后，Clamless 会请求 Apple Silicon 内置面板 framebuffer 进入低功耗关闭状态。

03

优先安全恢复

外接显示器消失时，Clamless 会优先恢复内置显示器，而不是信任可能过期的插拔状态。

安装

安装一次，从菜单栏使用。

下载最新 DMG，把 Clamless 拖进 Applications，然后启动。当前公开构建是 ad-hoc 签名，首次启动时 macOS 可能需要你右键点击应用并选择打开。

工作原理

两层显示系统，一个菜单栏开关。

SkyLight 布局控制

Clamless 使用私有 SkyLight 显示拓扑调用，在 active desktop arrangement 中 disable 或 enable 内置显示器。

IOMobileFramebuffer 面板电源

布局移除后，它会通过 Apple Silicon framebuffer API 请求面板电源状态变化，让内置面板变黑。

Unplug 优先恢复

自动恢复结合 1 秒轮询、显示器回调、IORegistry 事件和布局层安全检查，在外接显示器拔掉时恢复内置屏。

clamless-display status
clamless-display off --commit session
clamless-display on --commit session

内置 CLI

菜单不够时，也可以脚本化。

内置 helper 暴露了和菜单栏应用相同的显示控制路径，适合调试、自动化和检查当前显示状态。

现实边界

实用工具，不是官方接口。

Clamless 依赖 macOS 私有 API。正因为如此，它才能实现系统设置没有提供的行为；但 Apple 也可能在未来 macOS 版本中修改这些接口。

不收集 telemetry。

不是黑色遮罩窗口。

不是只把亮度调低。

不声称内置面板会从 IORegistry 消失。
